# /idea

Превращение идеи пользователя в фичу бэклога.

## Процесс

1. **Получить идею** от пользователя (может быть сырая, 1-2 предложения)

2. **Сохранить сырую идею** в `backlog/YYYY-MM-DD-идея.md`:
   ```markdown
   # Идея: [краткое название]
   Дата: YYYY-MM-DD
   Автор: пользователь
   
   ## Исходная формулировка
   [текст как сказал пользователь]
   
   ## Статус: Необработанная
   ```

3. **Вызвать product-analyst** для анализа:
   - Показать сырую идею
   - Задать уточняющие вопросы если нужно
   - Получить готовую фичу

4. **Добавить в бэклог** (`docs/BACKLOG.md`):
   - Вставить фичу в нужную категорию
   - Установить статус "Бэклог"
   - Если приоритет high → переместить в "Актуальные задачи"

5. **Обновить статус** сырой идеи на "Обработана → [ссылка на фичу]"

## Вопросы для уточнения (если идея сырая)

- Это для какого проекта/контекста?
- Кто будет пользователем?
- Какая главная выгода?
- Есть ли дедлайн?

## Когда вызывать

Когда пользователь говорит:
- "У меня есть идея..."
- "Нужно сделать..."
- "Хочу добавить..."
- Или любая неоформленная мысль о задаче

## Формат фичи на выходе

```
## [Категория] Название фичи

**Статус:** Бэклог | В работе | Готово
**Дедлайн:** YYYY-MM-DD (опционально)
**Приоритет:** high | medium | low

**Описание:**
Что это и зачем.

**Критерии готовности:**
- [ ] Критерий 1
- [ ] Критерий 2

**Связанные задачи:**
- backlog/YYYY-MM-DD-идея.md
```
