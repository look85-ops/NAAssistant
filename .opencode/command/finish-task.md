# /finish-task

Закрыть текущую задачу и зафиксировать результат.

## Процесс

1. Проверить критерии готовности: пройтись по чекбоксам в описании фичи. Если что-то не сделано — сообщить пользователю.
2. Обновить статус фичи на "Готово" в docs/BACKLOG.md и добавить дату закрытия.
3. Зафиксировать результат (кратко: что сделано, риски/заметки).
4. Закоммитить изменения: показать diff, подтвердить у пользователя, выполнить git add/commit. Формат: `[F-NNN] Название фичи`.
5. Если работаем не в main — открыть PR в main. Дождаться зелёного CI перед merge.
6. Предложить следующую приоритетную задачу: «Берём F-NNN?»

## Когда вызывать

Когда пользователь говорит: «Закрой задачу», «Готово, коммить», «Закончили, коммитим», фича выполнена.

## Формат коммита

```
[F-NNN] Название фичи

- Что сделано 1
- Что сделано 2
```
